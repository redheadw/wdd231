Site Purpose

Chat Connect provides a safe, friendly place for people to join chatrooms and connect through conversation. Visitors can register or log in, browse chatrooms, filter by location (within 250 miles), or join USA-wide rooms. The site will expand in the future to support chatting anywhere the service is available.

  • Home page: clear entry points (Register / Log In) and links to key info pages.
  • Chatrooms: discover rooms by local radius or USA-wide categories.
  • Info: About, FAQ, Privacy, Terms of Use (combined where needed for the 3-page requirement).

Dynamic Features and Data Plan

Data source (Fetch + JSON)

  • Use a local JSON file (example: data/chatrooms.json) with at least 15 chatrooms.
  • Each chatroom item shows at least 4 properties (example: name, topic, city/state, memberCount, safetyRating).
  • Use fetch() + async/await and try...catch.

Browser APIs

  • Geolocation API: for â€œLocal within 250 milesâ€ (user can allow/deny).
  • LocalStorage: remember filter choice (Local / USA) and preferred topics.

Required JS patterns

  • DOM selection with querySelector/querySelectorAll
  • Event listeners for filter controls, search input, and room cards
  • Array method usage: filter (local radius), map (render cards), forEach
  • Template literals to generate chatroom cards
  • ES Modules: separate files like chatrooms.js, storage.js, ui.js

Modal dialog requirement

  • Clicking a chatroom opens a modal with room details + â€œJoinâ€ button.
  • Modal supports keyboard escape, focus management, and click-outside close.

Performance plan

  • Optimize images with proper sizing, format, and loading="lazy".
  • Keep total page weight under 500kB from empty cache.

Complition plan
